# Academy Versioning

Course modules and SOPs carry a `version` field. Module completion is stored with `moduleVersion`, so historical completion remains intact after a new version is published.

Founder content edits mark modules as no longer seed-managed. When `incrementVersion` is checked in the admin editor, new completion is required for the new version.

Policies and acknowledgement text are stored with the module version to support re-acknowledgement when policy language changes materially.
